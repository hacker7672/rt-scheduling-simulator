How it Works 🚀

This simulator visually demonstrates how real-time operating systems schedule tasks using the CPU.

  • Add Tasks: Enter a Name, Execution Time C (time needed to complete), and Period T (interval between task arrivals).
  • RM (Rate Monotonic): Tasks with shorter periods (T) get higher priority (fixed).
  • EDF (Earliest Deadline First): Tasks with deadlines closest to the current time get highest priority (dynamic).
  • Deadlines: The vertical dashed lines indicate when a task MUST finish. If a task fails to finish on time, it pulses RED!
  • Preemption: A running task stops splitting its block if a higher priority task wakes up and needs CPU time.
